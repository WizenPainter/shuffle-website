# Shuffle — website

Marketing site for [Shuffle](https://github.com/WizenPainter/shuffle), a fast,
native, GPU-rendered file manager for macOS. Built with Next.js 16 (App Router)
+ Tailwind CSS v4. Fully static — deploys anywhere (Vercel, Netlify, GitHub
Pages, S3, …).

## Develop

```sh
npm install
npm run dev      # http://localhost:3000
npm run build    # static production build
```

## Demo videos

Web-optimized clips live in `public/videos/` (`hero`, `split`, `search`,
`customization`), each with an `.mp4` and a `-poster.jpg` first frame. They're
referenced centrally from `clips` in `src/lib/site.ts` and rendered by
`src/components/VideoFrame.tsx` (autoplay, loop, muted, `playsInline`). The
source recordings are kept in the top-level `videos/` directory.

To replace or add a clip, re-encode from a source recording (H.264, muted,
`+faststart`) — for example:

```sh
ffmpeg -i videos/my-recording.mp4 -an \
  -vf "scale=1600:-2:flags=lanczos" \
  -c:v libx264 -profile:v high -pix_fmt yuv420p -crf 26 -preset slow \
  -movflags +faststart public/videos/hero.mp4

# matching poster (first frame)
ffmpeg -ss 1 -i public/videos/hero.mp4 -frames:v 1 \
  -vf "scale=1200:-2" public/videos/hero-poster.jpg
```

Then update the matching entry in `clips` (`src/lib/site.ts`) if the filename
changed. A frame with no `src` falls back to a labelled "Video coming soon"
placeholder.

## The download (.dmg)

The Download buttons point at `public/Shuffle.dmg` (see `dmgUrl` in
`src/lib/site.ts`). To ship a new build, replace that file:

```sh
cp /path/to/Shuffle.dmg public/Shuffle.dmg
```

For large binaries you may prefer hosting the `.dmg` on a GitHub Release or a
CDN and pointing `dmgUrl` at that URL instead of bundling it.

## Editing content

All copy, features, shortcuts, FAQs, and links live in `src/lib/site.ts` —
GitHub URL, DMG path, feature list, keyboard shortcuts, and the FAQ Q&A. The
Free/Pro pricing block is in `src/components/OpenSource.tsx` (the one-time Pro
purchase is marked "Planned").

## SEO & AI discoverability (AISEO)

> **⚠️ Set the domain.** Everything below uses `site.url` in `src/lib/site.ts`
> (currently `https://shuffle.app`). Change that one value to the real
> production domain before launch — canonical URLs, the sitemap, robots, JSON-LD
> and `llms.txt` all derive from it.

Search engines:

- **Rich metadata** — title template, description, keywords, canonical,
  OpenGraph + Twitter cards, and robots directives (`src/app/layout.tsx`).
- **Structured data (schema.org JSON-LD)** in `src/components/JsonLd.tsx`:
  `SoftwareApplication` (with free `Offer`), `Organization`, `WebSite`, and a
  `FAQPage`. Validate at [Rich Results Test](https://search.google.com/test/rich-results).
- **`sitemap.xml`** and **`robots.txt`** are generated by `src/app/sitemap.ts`
  and `src/app/robots.ts`. A **web app manifest** comes from `src/app/manifest.ts`.
- Semantic headings, descriptive alt text, and a static (fast) build.

AI assistants (ChatGPT, Claude, Gemini, Perplexity, DeepSeek, …):

- **`public/llms.txt`** — a clean, factual, citation-friendly summary following
  the [llmstxt.org](https://llmstxt.org) convention. Keep it in sync with
  `site.ts` when facts change.
- **`robots.ts` explicitly allows AI crawlers** (GPTBot, OAI-SearchBot,
  ClaudeBot, PerplexityBot, Google-Extended, Applebot-Extended, DeepSeekBot,
  and more) so assistants may fetch and cite the site.
- The **on-page FAQ** (`src/components/Faq.tsx`) mirrors the `FAQPage` schema —
  natural-language Q&A is what both featured snippets and LLM answers extract.
- The JSON-LD `SoftwareApplication` gives models unambiguous structured facts
  (free, MIT, macOS 12+, Rust/GPUI, feature list, download URL).
